# 2026-06-11 - dnd5e Cream Sheets Deep Theming

## Summary

- Prepared `0.5.0` (feature release on top of the `0.4.0` custom stickers / i18n work).
- Implemented full cream deep-theming for dnd5e 5.x: palette variable remap, decoration hooks, and seven per-surface refinement layers; JS-side light-theme forcing; dnd5e2 windows carved out of generic shell theming.

## Motivation

Before 0.5.0 the Animal Island shell (cream paper panels, warm brown text) surrounded dnd5e windows but could not reach inside them. The dnd5e 5.x sheet system compiles its own CSS variables at load time and paints dark slate backgrounds on ability panels, skill lists, and item rows. ## Approach

Three layers, applied in order:

1. **Palette variable remap** (`--dnd5e-color-*`, `--dnd5e-background-*`, `--color-text-*`) â€” CSS custom properties redeclared on `:is(.dnd5e2, .dnd5e2-journal)` and `.dnd5e2.chat-message` so every internal rule that reads a system variable gets a cream or warm-brown value automatically.
2. **Decoration hooks** â€” island banner headers, cream filigree boxes, and warm-yellow section headers built from pseudo-elements and `background-image` gradient stacks so they degrade gracefully if the markup changes.
3. **Per-target refinements** â€” seven dedicated CSS layers (3.1â€“3.7), each scoped to the target's real root classes (`.dnd5e2.sheet.actor`, `.dnd5e2.sheet.item`, `.dnd5e2.compendium-browser`, `.roll-table-sheet`, `.chat-message`, `#hotbar`), every selector statically verified against `dnd5e.css` / `foundry2.css` / the system templates before landing.

**Light-theme forcing (JS):** When `enableDnd5eCreamTheme` is on, two `renderApplicationV2` hooks add the `themed theme-light` classes (and strip `theme-dark`): one on every `.dnd5e2` window element, one on the `#chat` / `#chat-popout` containers (dnd5e chat cards take their palette from the descendant selector `.themed.theme-light .dnd5e2.chat-message`, so the ancestor must carry the classes). This forces the dnd5e system's own light-branch variables before the CSS remap runs, preventing dark-mode fallback values from leaking through unpainted variables.

**dnd5e2 carve-out:** The generic window-shell theming block (`.app`, `.window-app`, `.application`) previously leaked into dnd5e2 windows and competed with the new sheet CSS. All `.dnd5e2` descendants are now excluded from the generic shell with `:not(.dnd5e2)` guards, so dnd5e2 windows are owned entirely by `styles/dnd-animal-sheets.css` and revert to fully native appearance when the cream theme is toggled off.

## Affected Surfaces

Seven refinement targets added in `styles/dnd-animal-sheets.css`:

1. **PC character sheet** (`.dnd5e2.sheet.actor.character`) â€” ability panel, skill rows, resources, biography tabs.
2. **NPC sheet** (`.dnd5e2.sheet.actor.npc`) â€” stat block header, trait pills, action/legendary sections.
3. **Item sheet** (`.dnd5e2.sheet.item`) â€” detail rows, activity list, advancement list.
4. **dnd5e chat cards** (`.chat-message .dnd5e2`) â€” dice total rings with crit (teal) and fumble (rose) variants.

## Settings Migration

`enableDnd5eSheetSafety` and `enableDnd5eAnimalBackground` (both world scope) are removed. They are replaced by a single `enableDnd5eCreamTheme` world toggle (default `true`). Old world-storage keys are orphaned silently â€” Foundry ignores unknown setting keys. No migration script needed.
